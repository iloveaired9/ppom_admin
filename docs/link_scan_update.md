# 비정상 링크 스캐너 업데이트 리포트 (2026-03-16)

## 1. 개요
링크스캔 로직에서 `href` 속성 맨 앞에 공백이 포함된 태그(`<a href=" //..." >`)를 비정상 링크로 감지하지 못하는 이슈를 해결하기 위해 로직을 고도화하고, 테스트 환경에도 동일한 기능을 적용했습니다.

## 2. 주요 변경 사항

### 2.1. 탐지 로직 강화 (Content Script)
기존의 단순 문자열 비교(`startsWith`) 방식에서 정규표현식(`test`) 방식으로 변경하여, 단순 공백뿐만 아니라 탭, 줄바꿈 없는 공백(`&nbsp;`) 등 모든 종류의 공백 문자를 완벽하게 탐지하도록 개선했습니다.

- **대상 파일:** `content_script.js`
- **핵심 코드:**
  ```javascript
  const isAbnormal = /^\s/.test(href) || // 공백 시작 체크
                     /^http:\/\//.test(trimmedHref) || 
                     (/^http:[^\/]/.test(trimmedHref) && !/^https:/.test(trimmedHref));
  ```

### 2.2. 테스트 도구 동기화
사용자가 `test/ppom_link` 디렉토리의 도구를 사용할 때도 동일한 수준의 검사 결과를 얻을 수 있도록 비정상 링크 스캔 기능을 추가하고 로직을 동기화했습니다.

- **대상 파일:** `test/ppom_link/content.js`
- **추가 기능:** 메인 익스텐션의 `scanForHttpLinks` 및 하이라이트/스크롤 로직 반영

## 3. 검증 결과

| 케이스 | 입력 태그 예시 | 탐지 결과 | 비고 |
| :--- | :--- | :---: | :--- |
| **일반 공백** | `<a href=" //ko.ali...">` | **성공** | `href.startsWith(' ')` 로도 탐지 가능 |
| **복수 공백** | `<a href="  //ko.ali...">` | **성공** | 개선된 정규식으로 탐지 |
| **특수 공백** | `<a href="\xA0//ko.ali...">` | **성공** | NBSP 등 특수 공백 탐지 가능 |
| **기존 패턴** | `<a href="http://...">` | **성공** | 기존 HTTP 탐지 유지 |

## 4. 확인 방법
1. 익스텐션 관리 페이지(`chrome://extensions`)에서 **'새로고침'** 클릭
2. 뽐뿌 또는 테스트 페이지 접속
3. 사이드패널의 **'링크스캔'** 탭에서 **'지금 스캔'** 실행
4. 리스트에 정상적으로 노출되는지 확인
